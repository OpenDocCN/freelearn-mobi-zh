Chapter 13. Deployment to Other Platforms

Continuing the talk about multiplatform development, we should spend some time analyzing what other platforms are supported by Moai SDK.

Moai SDK ships with example hosts for different platforms. It's recommended that you read and understand the code for the platform's hosts, so you are able to modify them. If you wish to add your own fancy advertising library, or integration with the next generation social network, you'll need to modify the host.

Let's take a look at the hosts that come in the Moai SDK release.

Windows

In order to build for Windows you need to use one of the Visual Studio projects. You'll find both of them inside moai-sdk/hosts/folder (vs2010 and vs2008).

Open up the correct solution for your Visual Studio version. Inside, you'll find all the code you need for the GlutHost, which should run smoothly on windows. Moai SDK already ships with a compiled version of this host in moai-sdk/bin/win32/.

Mac OS X

The host for Mac OS X can be compiled using the Xcode project from moai-sdk/hosts/xcode/osx.

The codebase for this host is basically the same as for the Windows one; it uses the GlutHost program with some considerations to make it work on Mac OS X.

Android

The Android hosts are generated using a shell script located in moai-sdk/hosts/android/run-host.sh (or .bat). Be sure to have Apache Ant installed and all the Android stuff as well. Take a look at the wiki page for more information regarding building your game in Android (http://getmoai.com/wiki/index.php?title=Building_Moai_Games_For_Android_Devices).

Note

Compiling for Android could be a whole chapter by itself. Besides the mentioned link in Moai SDK wiki, you'll also need some knowledge of Android development, so it's recommended that you head over and read Android's documents before trying to build the host (http://developer.android.com/develop/index.html).

Google Chrome (native client)

Native client (https://developers.google.com/native-client/) is a new way to introduce code that runs natively in the computer and that can be downloaded from your Chrome browser. Moai SDK comes with a host built for it, which can be found in moai-sdk/hosts/chrome.

You'll need to change the manifest.json file and put all your code in the same folder.

Detailed instructions on how to do this are located at Moai's wiki link (http://getmoai.com/wiki/index.php?title=Building_Moai_Games_for_Google_Chrome).

Linux

At the time of writing the book, Linux is not available as a target platform in the Moai SDK release, but Moai works on it building from the source code.

There are two main ways to do this right now:

  • Using the Linux branch from the Moai SDK's GitHub repository as explained at http://franciscotufro.com/2013/01/compiling-moai-on-linux/
  • Using the spacepluk's branch as explained at https://github.com/spacepluk/moai-dev

The second method is preferred for now since it has Untz support, but official Linux support is on the way, so double check the latest Moai SDK release to verify that Linux is not yet officially supported.
